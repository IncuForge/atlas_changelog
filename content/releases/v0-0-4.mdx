## Steadier model switching

Changing models in Compass now goes through Pi's JSON-RPC interface in a more consistent way, cutting down on the mismatches between what Compass shows and what Pi is actually running.

## Provider-aware thinking controls

Compass now surfaces thinking and reasoning controls that adapt to what the selected Pi provider actually supports, instead of presenting options that may not apply.

## Better provider discovery

Pi provider discovery and catalogue handling have been improved, making it easier for Compass to find configured providers and present an accurate, up-to-date list of models.

## More reliable packaged builds

Packaged application environments — the macOS, Windows, and Linux builds most people install — now resolve Pi and its providers more reliably, building on the discovery fixes from v0.0.3.

## Cleaning up the edges

Legacy settings routing has been simplified, and the obsolete telemetry-authoring skill has been removed along with an update to Compass's documentation and roadmap.
